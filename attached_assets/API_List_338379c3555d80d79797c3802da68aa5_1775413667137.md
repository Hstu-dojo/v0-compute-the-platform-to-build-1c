# Full API Routes & Request/Response Mocks

## Conventions

All routes are prefixed with `/api/v1`. All requests require `Authorization: Bearer <firebase_jwt>` unless marked `[public]`. All timestamps are ISO 8601 UTC. All errors follow a consistent shape. Pagination uses cursor-based pagination, not offset.

**Standard error shape:**

```json
{
  "error": {
    "code": "INSUFFICIENT_CREDITS",
    "message": "Not enough credits to complete this request.",
    "details": {
      "required": 450,
      "available": 120
    }
  }
}
```

**Standard paginated list shape:**

```json
{
  "data": [],
  "pagination": {
    "next_cursor": "eyJpZCI6Ii4uLiJ9",
    "has_more": true,
    "limit": 20
  }
}
```

## Group 1 — Authentication

---

### `POST /api/v1/auth/register`

Registers a new user after Firebase has issued the JWT. Creates the user record, profile, free plan subscription, and initial credit grant in one transaction.

**Request:**

```json
{
  "display_name": "Rahul Ahmed"
}
```

**Headers:**

```
Authorization: Bearer <firebase_jwt>
Content-Type: application/json
```

**Response `201`:**

```json
{
  "user": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "firebase_uid": "firebase_abc123",
    "email": "rahul@example.com",
    "display_name": "Rahul Ahmed",
    "role": "student",
    "is_email_verified": false,
    "created_at": "2025-04-04T10:00:00Z"

  "subscription": {
    "plan": "free",
    "status": "active",
    "monthly_credit_allotment": 500
  },
  "token_balance": 500

```

```json
{
  "error": {
    "code": "USER_ALREADY_EXISTS",
    "message": "An account with this Firebase UID already exists."
  }
}
```

Accepts a PDF file upload. Runs virus scan, stores to R2, extracts text, checks for duplicates by content hash, and enqueues the embedding job.

**Request:** `multipart/form-data`

```
file: <binary pdf>
title: "Organic Chemistry Chapter 4"
```

**Response `201`:**

```json
{
  "document": {
    "id": "doc_11223344-aabb-ccdd-eeff-001122334455",
    "title": "Organic Chemistry Chapter 4",

    "created_at": "2025-04-04T10:05:00Z"
```

## Group 4 — WebSocket: Batch Progress

---

### `WSS /ws/batch/:batch_id`

The client connects immediately after receiving the `202` from `/study-sets/generate`. The server pushes events as each worker finishes. The client listens and hits the corresponding output route when it receives a `job_completed` event.

**Connection handshake — client sends on open:**

```json
{
  "type": "auth",
  "token": "wss_ephemeral_token_xyz"
}
```

**Server acknowledges:**

```json
{
  "type": "connected",
  "batch_id": "batch_aabb1122-ccdd-3344-eeff-556677889900",
  "subscribed_at": "2025-04-04T10:15:02Z"
}
```

**Server event — job started:**

```json
{
  "type": "job_started",
  "job_id": "job_notes_uuid",
  "task_type": "notes",
  "timestamp": "2025-04-04T10:15:05Z"
}
```

**Server event — job completed:**

```json
{
  "type": "job_completed",
  "job_id": "job_notes_uuid",
  "task_type": "notes",
  "output_id": "out_notes_uuid",
  "credits_consumed": 287,
  "timestamp": "2025-04-04T10:15:42Z"
}
```

The client receives `job_completed`, extracts `output_id` and `task_type`, then calls the appropriate output route. For example, if `task_type` is `notes`, the client calls `GET /api/v1/study-sets/output/out_notes_uuid/notes`.

**Server event — job failed:**

```json
{
  "type": "job_failed",
  "job_id": "job_mcq_uuid",
  "task_type": "multiple_choice",
  "error": "AI provider unavailable after 3 retries",
  "credits_released": 150,
  "timestamp": "2025-04-04T10:16:10Z"
}
```

**Server event — job retrying:**

```json
{
  "type": "job_retrying",
  "job_id": "job_mcq_uuid",
  "task_type": "multiple_choice",
  "attempt_number": 2,
  "next_retry_at": "2025-04-04T10:15:55Z",
  "timestamp": "2025-04-04T10:15:45Z"
}
```

**Server event — batch complete:**

```json
{
  "type": "batch_completed",
  "batch_id": "batch_aabb1122-ccdd-3344-eeff-556677889900",
  "total_jobs": 4,
  "completed_jobs": 3,
  "failed_jobs": 1,
  "total_credits_consumed": 585,
  "timestamp": "2025-04-04T10:16:45Z"
}
```

Server closes the WebSocket connection after sending `batch_completed`.

**Client heartbeat ping — sent every 30s to keep connection alive:**

```json
{ "type": "ping" }
```

**Server pong:**

```json
{ "type": "pong" }
```

### `GET /api/v1/study-sets/output/:output_id/notes/stream`

Streams the generated notes in markdown format, chunk by chunk, exactly like ChatGPT's streaming response. Uses `text/event-stream` (SSE). Each event contains a delta of the markdown string. The client appends each delta to the display buffer in real time.

**Response headers:**

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```

**Stream events:**

First event — metadata:

```
event: metadata
data: {"output_id":"out_notes_uuid","model":"gpt-4o","provider":"openai"}
```

Subsequent events — markdown delta chunks as they stream from the model:

```
event: delta
data: {"delta":"# Organic Chemistry — Chapter 4\n\n"}

event: delta
data: {"delta":"## Key Concepts\n\n"}

event: delta
data: {"delta":"### 1. Alkyl Halides\n\nAlkyl halides are compounds where a halogen atom "}

event: delta
data: {"delta":"(F, Cl, Br, I) replaces a hydrogen on an alkane chain."}

event: delta
data: {"delta":" They are classified by the carbon bearing the halogen:\n\n"}

event: delta
data: {"delta":"- **Primary (1°)** — halogen-bearing carbon is attached to one other carbon\n"}
```

Final event — signals stream end with usage stats:

```
event: done
data: {"credits_consumed":287,"prompt_tokens":1840,"completion_tokens":920,"total_tokens":2760}
```

**How the client handles this:**

The client opens an `EventSource` to this URL. On each `delta` event it appends the `delta` string to a local string buffer and passes that buffer to a markdown renderer (e.g. `react-markdown`) which re-renders incrementally. On the `done` event, the client closes the `EventSource` and updates the token balance display.

**Errors during stream:**

```
event: error
data: {"code":"STREAM_INTERRUPTED","message":"Connection to AI provider lost mid-stream."}
```
